# Task: Season Image System (rank emblems + achievement badges)

## Original request

> "for season rank and achievements we need the same flow and style like we did for the artifacts. its need to be sheet generated by the agent in the end - when user asks to make new bitmap images, using deterministically approach, analyze it and reuse the code where its possible"

User-confirmed clarifications:

1. **One combined pipeline** for season rank emblems and run-achievement badges (single style guide, single workspace, single contact sheet, single metadata file).
2. **Keep the inline SVG / glyph fallback** during the mid-batch period; remove once every entry has approved provenance metadata (backlog item below).
3. **Land as a single commit** on the active branch — no separate toolkit-refactor PR.

## Acceptance criteria

- **AC1** — Asset-agnostic primitives extracted to `app/scripts/lib/bitmap-image-toolkit.js` (PNG decode, alpha analysis, hashing, sheet manifest tracking, metadata bundle, provenance check). The four artifact scripts (`validate-artifact-image-coverage`, `generate-artifact-contact-sheet`, `generate-artifact-image-metadata`, `check-artifact-image-provenance`) consume the toolkit instead of inlined copies.
- **AC2** — Existing artifact pipeline still functional after refactor: `npm run game:artifacts:validate -- --all`, `npm run game:artifacts:provenance:check`, and `npm run game:artifacts:sheet --validate-only` all pass with no regressions.
- **AC3** — Season pipeline has parallel scripts and npm aliases:
  - `game:season:next` → next batch of imagegen prompts (skips entries whose PNG already exists)
  - `game:season:validate` → coverage / bbox / edge / freshness checks
  - `game:season:sheet` → deterministic Puppeteer contact sheet with manifest
  - `game:season:provenance:generate` → writes `app/shared/season-image-metadata.json`
  - `game:season:provenance:check` → asserts every metadata sha256 matches the file on disk
- **AC4** — Production output convention: `web/public/season-ranks/{rank}.png` (4 entries) and `web/public/achievements/{id}.png` (21 entries — 9 general + 12 character). Workspace path: `.agent/season-image-workspace/{raw,processed,review}/`, gitignored.
- **AC5** — Style guide and todolist docs in place: `docs/season-image-style-prompt.md`, `docs/season-image-todolist.md`. The todolist is parsed by the `next` script for per-id descriptions.
- **AC6** — Runtime swap: `SeasonRankEmblem.js` tries the PNG and falls back to the existing inline SVG; new `AchievementBadge.js` component tries the PNG and falls back to the existing emoji glyph. `console.warn` once per missing entry. Frontend bundle still builds clean.
- **AC7** — `npm run game:test` (371 tests) passes after the refactor.

## Constraints

- No new native dependencies (PNG decode stays pure-JS, Puppeteer reused).
- Don't break the artifact pipeline: every artifact script must still produce byte-identical output for the same inputs.
- Don't leak workspace paths into version control.
- The provenance metadata file is the only thing the runtime is allowed to trust as "approved".

## Non-goals

- Generating the actual production PNGs in this task. Image generation is a manual imagegen step driven by `npm run game:season:next`.
- Replacing the existing emoji glyph badges visually — that happens *after* every PNG ships and gets approved provenance.

## Verification plan

- `npm run game:test` — backend + composables pass.
- `npm run game:build` — frontend bundle builds clean.
- `npm run game:artifacts:provenance:check` — refactor preserves existing approved metadata.
- `npm run game:artifacts:validate -- --all` — all 41 artifacts still pass coverage rules.
- `node app/scripts/generate-artifact-contact-sheet.js --validate-only` — DOM validation still passes.
- `npm run game:season:next --limit=2` — prompts print correctly.
- `npm run game:season:validate -- --all` — fails cleanly with "missing PNG" for every entry (expected pre-asset-generation state).
